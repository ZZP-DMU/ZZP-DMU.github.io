Hexo快速入门

概要

本文介绍了通过hexo + github page搭建静态博客过程中的步骤和遇到的问题。请注意： 我部署到github page上的只有静态html文件，没有源码！至于什么是静态html文件和源码见目录结构。我参考的内容是官方文档中的一键部署

依赖

  • git
  • node.js

因为我的电脑之前已经配置过了，所以我在依赖方面没有遇到障碍。关于依赖的安装和版本要求见官网文档.

安装

官方文档给出的安装步骤如下：

npm install hexo-cli -g
hexo init blog
cd blog
npm install
hexo server

其实还是非常简单的。但是还是要具体讲一下细节和我遇到的问题：

  • npm install hexo-cli -g命令在我电脑上是需要sudo来提权的。
  • 首先在工作新建一个目录mkdir -v hexo,然后在目录hexo内进行hexo init blog命令，这个命令会在hexo目录下新建blog目录。
  • hexo init blog命令在我电脑上失败了，但是cd进blog目录后执行npm install命令就可以解决问题。
  • 执行hexo server就能在本地4000端口访问了，如果端口冲突，可以使用-p <port-number>来解决。
    当然还是有一些高阶的安装方法，但是本着tldr的原则，我选择忽略。

目录结构

可能是我能力不行吧，我觉得官网的文档写得非常非常非常糟糕。下面讲讲我对hexo的认识吧：

1.首先，讲讲hexo serverorhexo s命令吧，这是一个在本地开启服务的命令，相信在安装部分读者已经尝试过这个命令了。该命令是不生成任何输出文件的。

2.其次，如果我们想要将我们的博客系统部署到github上，应该怎么做呢？答案是使用hexo generateorhexo g命令，它的作用是生成能够直接解析的html文件。
该命令成功运行后将在hexo\blog目录下生成一个public文件夹，我们只需要将这个文件夹的内容部署到github上就可以在外网访问了，部署详细见部署到github。

3.那么，blog目录下还有什么呢？我认为最重要的就是source & themes & _config.yml了。

4.其中source就是我们的markdown格式的文章所存储的地方，也就是源文件（源码）的一部分了。通常我们不需要手动在source目录下添加文件，hexo new post命令就可以在source\_post目录下添加我们的文章了。那么_post存储的是我们博客能被看到的部分，而source目录下像_drafts内的文章是不可见的，类似草稿。现在我们不需要弄清出这么多，详细可见官方文档。

5.themes存放的使我们博客主题的源码内容。在没有安装自定义主题时，该目录下只有一个空的.gitkeep文件（我的是这样）。关于主题的详细内容见主题安装

6._config.yml是最重要的配置文件了！但是我们在这里不说，因为tldr!有关内容可以见部署到github和主题安装

部署到github

笔者在部署到github上算是遇到不少坑。读者们千万小心！

详细步骤我就不说了RTFM&STFW，见官网一键部署,但是有几点要注意：

  • 首先，去github新建一个仓库。是的这一步很重要,其中最重要的就是仓库的名称了，一定一定要是<usr-name>.github.io!否则将来的网址会非常复杂，且有各种各样奇怪的问题。
  • 其次在_config.yam中的相关配置务必如下：
deploy:
  type: 'git'
  repo: git@github.com:<usr-name>/<usr-name>.github.io.git 
  branch: master

url: https://<usr-name>.github.io/
  • 确保能使用git登录github!我使用的ssh秘钥，如果和我一样则可以使用ssh -T git@github.com来检查git是否能登录github。

主题安装

不同的主题有不同的配置方法，所以我建议选择中文的主题，这样可以快速上手配置。

我使用的是white主题，其源码地址如下：white，模板地址：white，主题文档：主题文档。

这个主题在_config.yml中的配置也是最简单的，只需要添加theme: white即可！上手是非常快的！
