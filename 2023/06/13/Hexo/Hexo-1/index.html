Hexo进阶配置

markdown图片

_config.yam配置如下：

post_asset_folder: true
marked:
    prepenRoot: true
    postAsset: true

进行以上配置之后，每次我们使用hexo new post <article>创建新文章时，都会同时在source目录下创建同名的目录，用以储存该文章所要用到的图片资源。部署到github上是亲测好用的。

内网穿透，告别github

如果我们本地有内网穿透的话，只要内网穿透的类型为HTTPS，且端口号为<port-number>,那么我们就可以使用hexo s -p <port-number>,这样我们使用内网穿透的域名就能访问我们的博客系统了！

虽然我们告别了github，但是我们又陷入了内网穿透限制带宽的深渊了，如何选择就要看读者的口袋了。

_posts目录的整理

如果未来我们的博客系统变得庞大起来，博文的管理肯定会成为一个难题，所以我们应该对_posts目录下的文件进行管理。官方文档中并没有给出相应的办法，但是经过试验，我们可以直接在_posts目录下新建文件夹，将相同category的文件放在同一个文件夹内，同时注意将同名资源文件夹也要放进来。新建的文件没有办法自动在我们定义的文件夹中生成，但我们可以手动移动，对系统没有影响。

Hexo中自定义HTML

我的目的是能够在我的访问我的站点时，能够打开一张我自己定义的html页面。

我使用的主题是white,其他主题的配置方法和我的可能有所区别

  • 首先使用hexo new page slide命令创建一个新的页面，之后我将会把这个页面放在主页的菜单栏中
  • 在source目录中找到slide目录，并在其中新建一个lib目录，接下来我们自定义的html文件都将放在这个目录下
  • 在_config.yml中,配置为skip_render: "slide/lib/*"。因为这个配置选项只在source目录下起作用，所以我们不要配置成：skip_render: "/source/slide/lib/**",这样就会发生错误。这个配置是为了不让hexo渲染/slide/lib目录下的html文件，所以hexo中的格式不会对lib目录下的文件有影响。
  • 最后在white主题的配置文件_config.yml下，在menu条目下增加这么一条SLIDE: /slide，总得看是这样：

    menu:
    Home: /
    BLOG: /archives
    SLIDE: /slide

配置这个文件的作用就是在首页的菜单栏目增加我们刚才生成的slide页面，在source/slide/index.md中，我们可以创建链接指向lib目录下的自定义html文件

Hexo中实现对latex的支持

很简单，我们只需要执行以下命令：

  • cd blog
  • npm install hexo-math
  • 在需要插入公式的地方用以下格式
{% mathjax  %}
#your math equation in latex format
{% endmathjax %}
  • 插入行内公式可以用上面的mathjax，而如果想要插入多行公式就需要使用以下格式：
{% katex %}
#your math equation in latex format \\
#you can type here \\
#here too
{% endkatex %}

但是这个插件真的很难用

Hexo插入pdf

md中插入公式，特别是插入特别多的公式时，非常非常不好用，所以如果能够在hexo中插入pdf，我们就可以使用latex写完文章再嵌入hexo中就可以了。

  • 安装npm install hexo-pdf --save
  • 使用{% pdf your-pdf-path %}
